# 牛马回家 — 核心机制设计文档

## 1. 墙体（Wall）

### 定义
`label: 'wall'` 的静态刚体。在 Phaser Matter.js 中与地面（`ground`）一样阻挡玩家和墨水实体，但**不阻挡传送门射线**。

### 行为
| 交互对象 | 是否被墙阻挡 |
|---|---|
| 玩家移动 | ✅ 是 |
| 墨水实体 | ✅ 是 |
| 传送门射线 | ✅ 是 |

### 实现细节
墙体使用 `label: 'wall'` 的静态体。`PortalSystem._raycast()` 不跳过墙体，因此射线会被墙挡住，和地面/平台一样。

### 关键约束（平台游戏几何）
在 2D 横板游戏中，**地面到地面的近水平射线几乎无法越过垂直墙**——玩家站姿射击角度太平，任何拦路的墙都会挡住射线。

解决方案：**利用跳跃高度**。
- 玩家跳跃速度 JUMP=-14，重力 1.5，最大跳高 ≈ **65px**
- 墙高设为 **70px**：走路/普通跳无法越过；跳跃顶点（玩家升高约 105px）时俯角射击**恰好越过墙顶**
- 谜题要求：**跳起来在最高点右键射击橙色传送门**

### 关卡设计含义
- 传送门谜题区域（坑/宽沟）不设墙：宽度本身就是障碍
- 墙用于地面上的独立挡板谜题，配合"需要跳跃才能射"的技能要求
- 墙高 70px = 恰好比跳高（65px）多 5px，这个阈值要维持

## 2. 传送门视野限制（View-Limit）

### 规则
右键射出传送门时，**落点必须在当前相机视口内**（含 60px 缓冲边距）。超出屏幕范围的射击会被取消，并给出红色相机闪烁反馈。

### 实现细节
`PortalSystem._shoot()` 在 raycast 得到落点后立即检查：
```js
const margin = 60;
if (hit.x < cam.scrollX - margin || hit.x > cam.scrollX + cam.width + margin ||
    hit.y < cam.scrollY - margin || hit.y > cam.scrollY + cam.height + margin) {
  cam.flash(140, 160, 30, 30);   // 红色提示：超出视野
  return;
}
```

### 设计含义
- 玩家**不能隔屏盲射**，必须走近目标表面
- 与墙体结合：墙挡住视线 → 玩家移动到能看见墙后方地面的位置 → 穿墙射出传送门
- 给关卡设计提供额外约束：远处的传送台要求玩家先"移步看台"

## 3. 第一关传送门谜题结构

### 谜题 A — 坑 2（x = 562–800，238px 宽）
- 蓝色传送台：x = 498（小岛中心）
- 橙色传送台：x = 900（平台 3）
- **无墙**：坑本身宽到无法跳越，强制使用传送门
- 解法：在小岛上射蓝色（落地）→ 在平台 3 射橙色（落地）→ 走进蓝圈传送

### 谜题 B — 终点矮墙（x = 2350）
- 蓝色提示台：x = 2290（墙前，站着就能射）
- 橙色提示台：x = 2530（墙后，**必须跳起来射**）
- **挡板墙**：x = 2350，高 **70px**（恰好比最大跳高多 5px）
- 解法：跳起来在顶点右键射橙色 → 站着射蓝色 → 走进蓝圈传送到墙后 → 走向终点
